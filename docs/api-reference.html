Securaa API Reference

Comprehensive API documentation for the Securaa cybersecurity platform

API Overview

The Securaa platform provides comprehensive RESTful APIs for all components including SIA, SOAR, TIP, and CSAM. These APIs enable seamless integration with third-party tools, custom applications, and automated workflows.

Base URL: https://your-securaa-instance.com/api/v1
Authentication: API Key / OAuth 2.0
Format: JSON
Rate Limiting: 1000 requests/hour (configurable)

Authentication

API Key Authentication

Include your API key in the Authorization header:

curl -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     https://your-securaa-instance.com/api/v1/assets

OAuth 2.0 Authentication

For more secure applications, use OAuth 2.0 with client credentials flow:

# Get access token
curl -X POST https://your-securaa-instance.com/api/oauth/token \
     -H "Content-Type: application/x-www-form-urlencoded" \
     -d "grant_type=client_credentials" \
     -d "client_id=YOUR_CLIENT_ID" \
     -d "client_secret=YOUR_CLIENT_SECRET"

# Use access token
curl -H "Authorization: Bearer ACCESS_TOKEN" \
     -H "Content-Type: application/json" \
     https://your-securaa-instance.com/api/v1/threats

API Key Management

Generate API Key

POST /api/v1/auth/keys
{
  "name": "Integration API Key",
  "description": "Key for SIEM integration",
  "permissions": ["read:assets", "write:incidents"],
  "expires_in": 365
}

Key Permissions

  • read:assets - Read asset information
  • write:assets - Create/update assets
  • read:threats - Access threat intelligence
  • write:incidents - Create/update incidents
  • execute:playbooks - Execute SOAR playbooks
  • admin:* - Full administrative access

CSAM (Asset Management) API

Asset Operations

GET /api/v1/assets Retrieve all assets

Query Parameters

type string Filter by asset type (server, workstation, mobile, etc.)
criticality string Filter by criticality level (critical, high, medium, low)
status string Filter by asset status (active, inactive, retired)
limit integer Number of results to return (default: 50, max: 1000)
offset integer Pagination offset (default: 0)

TIP (Threat Intelligence) API

Indicators of Compromise (IoCs)

GET /api/v1/indicators Retrieve threat indicators

Query Parameters

type string Indicator type (ip, domain, hash, url, email)
confidence integer Minimum confidence level (0-100)
active boolean Filter active/inactive indicators
tags string Comma-separated tags

Webhooks & Real-time Events

Webhook Configuration

Configure webhooks to receive real-time notifications about security events:

POST /api/v1/webhooks Create webhook subscription 
{
  "url": "https://your-app.com/webhooks/securaa",
  "events": [
    "incident.created",
    "incident.updated",
    "threat.detected",
    "vulnerability.critical"
  ],
  "secret": "webhook_secret_key",
  "active": true
}

Supported Events

  • incident.created - New security incident
  • incident.updated - Incident status change
  • incident.resolved - Incident resolution
  • threat.detected - New threat indicator
  • vulnerability.critical - Critical vulnerability found
  • asset.discovered - New asset discovered
  • playbook.completed - Playbook execution finished
  • compliance.violation - Compliance violation detected

Webhook Payload Example

{
  "event": "incident.created",
  "timestamp": "2024-01-15T12:30:00Z",
  "data": {
    "incident_id": "incident_12345",
    "title": "Critical Vulnerability Detected",
    "severity": "critical",
    "assets": ["asset_67890"],
    "details": {
      "vulnerability": "CVE-2024-0001",
      "cvss_score": 9.8
    }
  }
}

WebSocket Real-time API

Connect to real-time event stream via WebSocket:

const ws = new WebSocket('wss://your-securaa-instance.com/api/v1/stream');

ws.onopen = function() {
    // Authenticate
    ws.send(JSON.stringify({
        type: 'auth',
        token: 'YOUR_API_KEY'
    }));
    
    // Subscribe to events
    ws.send(JSON.stringify({
        type: 'subscribe',
        events: ['incident.*', 'threat.detected']
    }));
};

ws.onmessage = function(event) {
    const data = JSON.parse(event.data);
    console.log('Received event:', data);
};

Error Handling

HTTP Status Codes

Code Status Description
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Invalid request parameters
401 Unauthorized Invalid or missing authentication
403 Forbidden Insufficient permissions
404 Not Found Resource not found

Security Best Practices

API Key Security

  • Store API keys securely (environment variables, key management systems)
  • Rotate keys regularly (recommended: every 90 days)
  • Use minimal required permissions
  • Monitor key usage and audit logs

Network Security

  • Use HTTPS for all API communications
  • Implement IP whitelisting where possible
  • Use VPN or private network connections
  • Validate SSL certificates

Request Security

  • Validate and sanitize all input data
  • Implement proper error handling
  • Use request signing for sensitive operations
  • Log and monitor API usage patterns
