CSAM Administration Guide

Comprehensive administration guide for Cyber Security Asset Management

Overview

The CSAM (Cyber Security Asset Management) administration interface provides comprehensive tools for managing your organization's digital and physical assets, vulnerability assessments, compliance monitoring, and security posture management.

Administrator Dashboard

Asset Summary

Real-time overview of all managed assets with categorization by type, criticality, and status

  • Total asset count and distribution
  • Asset type breakdown (servers, workstations, mobile, IoT)
  • Risk level categorization
  • Asset lifecycle status tracking

Vulnerability Overview

Critical vulnerability metrics and remediation status across your asset inventory

  • Critical and high-risk vulnerability counts
  • CVSS score distribution
  • Remediation progress tracking
  • SLA compliance monitoring

Compliance Status

Real-time compliance monitoring against industry standards and frameworks

  • NIST Cybersecurity Framework compliance
  • ISO 27001/27002 alignment
  • PCI DSS compliance status
  • Custom compliance frameworks

Discovery Status

Asset discovery scan status and automated inventory management

  • Active discovery scans
  • Last scan results and timestamps
  • Network coverage statistics
  • Discovery accuracy metrics

Asset Discovery Configuration

Discovery Methods

Network Discovery

Automated network scanning for asset identification

# Configure network ranges
Network Ranges: 10.0.0.0/8, 192.168.0.0/16, 172.16.0.0/12
Scan Ports: 22, 80, 135, 139, 443, 445, 3389
Discovery Schedule: Daily at 2:00 AM
Scan Intensity: Adaptive (based on network load)

Agent-Based Discovery

Deploy lightweight agents for detailed asset information

# Agent deployment
Agent Package: securaa-agent-v2.1.3
Supported OS: Windows, Linux, macOS
Collection Interval: 15 minutes
Data Points: Hardware, Software, Patches, Configurations

Criticality Levels

Critical

Mission-critical assets whose failure would cause significant business impact

  • Production database servers
  • Domain controllers
  • Payment processing systems
  • Core network infrastructure

High

Important assets that support business operations

  • Application servers
  • Email servers
  • File servers
  • Executive workstations

Medium

Standard business assets with moderate impact if compromised

  • Standard workstations
  • Development systems
  • Testing environments
  • Office printers

Low

Non-critical assets with minimal business impact

  • Personal devices
  • Guest networks
  • Demonstration systems
  • Archive servers

Vulnerability Management

Vulnerability Assessment Configuration

Scan Configuration

# Vulnerability Scan Settings
scan_profiles:
  full_scan:
    frequency: weekly
    depth: comprehensive
    plugins: all
    credentialed: true
  
  compliance_scan:
    frequency: monthly
    standards: [CIS, NIST, OWASP]
    remediation_tracking: true
  
  critical_asset_scan:
    frequency: daily
    targets: critical_assets
    severity_threshold: medium
    immediate_notification: true

Optimization Recommendations

Scan Optimization

  • Schedule intensive scans during off-peak hours
  • Use incremental scanning for frequently changing assets
  • Implement adaptive scanning based on asset criticality
  • Configure scan throttling to minimize network impact

Database Optimization

  • Regular database maintenance and indexing
  • Archive old vulnerability scan data
  • Optimize queries for large asset inventories
  • Implement data retention policies

Resource Scaling

  • Monitor resource usage trends
  • Scale compute resources based on asset count
  • Implement load balancing for high availability
  • Plan for peak scanning periods
