Deployment Guide

Step-by-step guide for deploying Securaa in production environments

Before You Begin

Important: Ensure all system requirements are met and hardware specifications are reviewed before proceeding with deployment.

Deployment Overview

This guide covers the deployment of Securaa platform components in various environments, from single-node deployments to large-scale enterprise configurations.

Single Node Deployment

Ideal for testing, development, or small environments

  • All components on single server
  • Minimal resource requirements
  • Quick setup and configuration

Multi-Node Deployment

Recommended for production environments

  • Distributed component architecture
  • High availability and scalability
  • Load balancing and redundancy

Cloud Deployment

Scalable cloud-native deployment

  • Container orchestration
  • Auto-scaling capabilities
  • Cloud-native integrations

Pre-Deployment Configuration

1. Environment Preparation

# Create deployment directory
sudo mkdir -p /opt/securaa/deploy
cd /opt/securaa/deploy

# Set proper permissions
sudo chown -R securaa:securaa /opt/securaa
sudo chmod -R 755 /opt/securaa

2. Configuration Files

Prepare the main configuration file:

# Copy template configuration
cp /opt/securaa/config/securaa.conf.template /opt/securaa/config/securaa.conf

# Edit configuration
nano /opt/securaa/config/securaa.conf

3. Database Configuration

Parameter Description Default Value
DB_HOST Database server hostname localhost
DB_PORT Database server port 5432
DB_NAME Database name securaa
DB_USER Database username securaa_user
DB_SSL Enable SSL connection true

Single Node Deployment

Step 1: Start Core Services

# Start database
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Start Redis cache
sudo systemctl start redis
sudo systemctl enable redis

# Start Elasticsearch
sudo systemctl start elasticsearch
sudo systemctl enable elasticsearch

Step 2: Deploy Application Components

# Deploy SIA component
/opt/securaa/bin/deploy-sia.sh --config /opt/securaa/config/securaa.conf

# Deploy SOAR component
/opt/securaa/bin/deploy-soar.sh --config /opt/securaa/config/securaa.conf

# Deploy TIP component
/opt/securaa/bin/deploy-tip.sh --config /opt/securaa/config/securaa.conf

# Deploy CSAM component
/opt/securaa/bin/deploy-csam.sh --config /opt/securaa/config/securaa.conf

Step 3: Start Securaa Services

# Start all Securaa services
sudo systemctl start securaa-sia
sudo systemctl start securaa-soar
sudo systemctl start securaa-tip
sudo systemctl start securaa-csam
sudo systemctl start securaa-web

# Enable auto-start on boot
sudo systemctl enable securaa-sia
sudo systemctl enable securaa-soar
sudo systemctl enable securaa-tip
sudo systemctl enable securaa-csam
sudo systemctl enable securaa-web

Multi-Node Deployment

Node Configuration

Management Node

  • Web interface
  • API gateway
  • Load balancer
  • Configuration management

Processing Nodes

  • SIA analytics engine
  • SOAR automation engine
  • TIP processing engine
  • CSAM scanning engine

Data Nodes

  • Primary database
  • Elasticsearch cluster
  • Redis cluster
  • File storage

Cluster Configuration

# Configure cluster on management node
/opt/securaa/bin/cluster-init.sh \
  --management-node \
  --cluster-name "securaa-prod" \
  --nodes "node1.securaa.local,node2.securaa.local,node3.securaa.local"

# Join processing nodes to cluster
/opt/securaa/bin/cluster-join.sh \
  --processing-node \
  --management-host "mgmt.securaa.local" \
  --node-type "sia,soar"

# Configure data nodes
/opt/securaa/bin/cluster-join.sh \
  --data-node \
  --management-host "mgmt.securaa.local" \
  --services "postgresql,elasticsearch,redis"

Cloud Deployment

Kubernetes Deployment

# Create namespace
kubectl create namespace securaa

# Deploy using Helm chart
helm repo add securaa https://charts.securaa.com
helm install securaa securaa/securaa \
  --namespace securaa \
  --set global.storageClass=fast-ssd \
  --set global.ingress.enabled=true \
  --set global.monitoring.enabled=true

Docker Compose Deployment

# Clone deployment repository
git clone https://github.com/securaa/deployment.git
cd deployment/docker

# Configure environment
cp .env.template .env
nano .env

# Deploy stack
docker-compose up -d

Security Configuration

SSL/TLS Configuration

# Generate SSL certificates
/opt/securaa/bin/generate-certs.sh \
  --domain "securaa.company.com" \
  --country "US" \
  --state "California" \
  --city "San Francisco" \
  --org "Your Company"

# Configure SSL in Nginx
sudo nano /etc/nginx/sites-available/securaa-ssl.conf
