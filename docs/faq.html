Installation & Setup

What are the minimum system requirements for Securaa?

The minimum requirements vary by deployment size:

Environment CPU RAM Storage Network
Small (1-100 assets) 4 cores 16 GB 500 GB SSD 1 Gbps
Medium (100-1000 assets) 8 cores 32 GB 1 TB SSD 1 Gbps
Large (1000+ assets) 16+ cores 64+ GB 2+ TB SSD 10 Gbps

For detailed specifications, see our Hardware Specifications guide.

Which operating systems are supported?

Securaa supports the following operating systems:

  • Linux: Ubuntu 20.04+, CentOS 8+, RHEL 8+, Debian 10+
  • Container: Docker 20.10+, Kubernetes 1.20+
  • Cloud: AWS, Azure, GCP, Oracle Cloud
  • Virtualization: VMware vSphere 6.7+, Hyper-V 2019+

Windows Server support is available through containerized deployment. See the Installation Guide for platform-specific instructions.

How long does installation typically take?

Installation time depends on your environment and configuration:

  • Single-node deployment: 30-60 minutes
  • Multi-node cluster: 2-4 hours
  • Cloud deployment (automated): 45-90 minutes
  • Air-gapped environment: 4-8 hours

This includes system preparation, software installation, and initial configuration. Additional time may be needed for integrations and customization.

Can I upgrade from a previous version?

Yes, in-place upgrades are supported with some considerations:

  • Supported upgrade paths: v2.0+ to current version
  • Backup required: Always create a full backup before upgrading
  • Downtime: Typically 15-30 minutes for small deployments
  • Rolling upgrades: Available for multi-node clusters

Review the upgrade documentation and test in a non-production environment first.

Configuration & Administration

How do I add users and manage permissions?

User management is handled through the administration interface:

  1. Navigate to Administration → User Management
  2. Click "Add User" and fill in the required information
  3. Assign appropriate roles: Administrator, Analyst, Viewer, etc.
  4. Configure multi-factor authentication (recommended)

You can also integrate with LDAP/Active Directory for centralized user management. See the Administration Guide for details.

How do I configure SSL/TLS certificates?

SSL/TLS configuration depends on your deployment method:

# For direct installation
sudo cp your-certificate.crt /opt/securaa/ssl/
sudo cp your-private-key.key /opt/securaa/ssl/
sudo chown securaa:securaa /opt/securaa/ssl/*

# Update configuration
sudo nano /opt/securaa/config/ssl.yml

# Restart web service
sudo systemctl restart securaa-web

For load balancer deployments, configure SSL termination at the load balancer level. Let's Encrypt certificates are supported for automatic renewal.

How do I backup and restore Securaa data?

Regular backups are essential for data protection:

# Manual backup
sudo /opt/securaa/bin/backup.sh --full --output /backup/securaa-$(date +%Y%m%d).tar.gz

# Automated daily backup (crontab)
0 2 * * * /opt/securaa/bin/backup.sh --incremental --output /backup/daily/

# Restore from backup
sudo /opt/securaa/bin/restore.sh --file /backup/securaa-20240115.tar.gz

Backups include configuration files, database, and user-uploaded content. Test restore procedures regularly.

How do I monitor system health and performance?

Securaa provides multiple monitoring options:

  • Built-in Dashboard: System health metrics in the web interface
  • API Endpoints: /api/v1/health and /api/v1/metrics
  • Log Files: Structured logging in /var/log/securaa/
  • SNMP: Integration with network monitoring systems
  • External Tools: Prometheus, Grafana, Nagios compatible

Set up alerting for critical metrics like CPU, memory, disk space, and service availability.

Features & Functionality

What's the difference between SIA, SOAR, TIP, and CSAM?

Each component serves a specific security function:

SIA (Security Intelligence Analytics)

Provides advanced analytics, threat detection, and security metrics across your environment.

SOAR (Security Orchestration)

Automates incident response workflows and orchestrates security tools through playbooks.

TIP (Threat Intelligence Platform)

Manages threat intelligence feeds, indicators of compromise, and threat actor information.

CSAM (Cyber Security Asset Management)

Discovers, classifies, and manages security posture of all organizational assets.

All components work together as an integrated platform for comprehensive cybersecurity operations.

Security & Compliance

What compliance frameworks does Securaa support?

Securaa includes built-in compliance templates and monitoring:

  • NIST Cybersecurity Framework: Complete implementation guidance
  • ISO 27001/27002: Information security management controls
  • PCI DSS: Payment card industry security standards
  • HIPAA: Healthcare information privacy and security
  • SOX: Sarbanes-Oxley financial reporting controls
  • GDPR: European data protection regulation
  • SOC 2: Service organization controls
  • CIS Controls: Center for Internet Security benchmarks

Custom compliance frameworks can be configured to meet specific organizational requirements.

How is data encrypted and protected?

Securaa implements comprehensive data protection:

  • Encryption at Rest: AES-256 encryption for all stored data
  • Encryption in Transit: TLS 1.3 for all network communications
  • Database Encryption: Transparent data encryption (TDE)
  • Key Management: Hardware security module (HSM) support
  • Access Controls: Role-based permissions and MFA
  • Audit Logging: Comprehensive activity tracking
  • Data Masking: Sensitive data protection in reports

All encryption keys are automatically rotated and can be managed through external key management systems.

Can Securaa be deployed in air-gapped environments?

Yes, Securaa supports completely offline deployments:

  • Local Repositories: Internal package and update mirrors
  • Manual Updates: Secure update process for isolated environments
  • Internal Intelligence: Threat intelligence generation from internal data
  • Custom Integrations: API-based integration with internal tools

Contact our team for air-gapped deployment planning and offline installation media.

Troubleshooting & Support

The web interface is slow or unresponsive. What should I do?

Performance issues can be caused by several factors:

  1. Check system resources: Monitor CPU, memory, and disk usage
  2. Review database performance: Look for slow queries or connection issues
  3. Clear browser cache: Force refresh and clear cached data
  4. Check network connectivity: Verify network latency and bandwidth
  5. Review recent changes: Consider recent configuration changes or updates

See our Troubleshooting Guide for detailed diagnostic procedures.

API calls are returning errors. How do I troubleshoot this?

API issues are often related to authentication or configuration:

# Test API connectivity
curl -I https://your-securaa-instance.com/api/v1/health

# Validate API key
curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://your-securaa-instance.com/api/v1/auth/validate

# Check API logs
tail -f /var/log/securaa/api.log

Common issues include expired API keys, insufficient permissions, rate limiting, or network connectivity problems. Check the API Reference for detailed error codes.

Threat intelligence feeds aren't updating. What's wrong?

Feed update issues can have several causes:

  • Check feed status: Verify feed health in the TIP administration panel
  • Test connectivity: Ensure external feed providers are reachable
  • Verify credentials: Check API keys and authentication for external feeds
  • Review rate limits: Ensure you're not exceeding provider rate limits
  • Check formatting: Verify feed data formats match expected schemas

Feed update logs are available in /var/log/securaa/tip-feeds.log for detailed diagnostics.

What are the recommended best practices for deployment?

Follow these best practices for optimal Securaa deployment:

  • Environment Separation: Use separate instances for development, testing, and production
  • High Availability: Deploy in clustered configuration for critical environments
  • Regular Backups: Implement automated daily backups with tested restore procedures
  • Security Hardening: Follow security hardening guidelines and keep systems updated
  • Monitoring: Set up comprehensive monitoring and alerting
  • Documentation: Maintain current documentation of configurations and procedures
  • Change Management: Use proper change control for configuration updates
  • User Training: Ensure all users receive appropriate training

How often should I update Securaa?

Update frequency depends on your environment and risk tolerance:

  • Security Updates: Apply immediately (within 24-48 hours)
  • Bug Fixes: Apply within 1-2 weeks after testing
  • Feature Updates: Quarterly or as business needs require
  • Major Versions: Plan upgrades 6-12 months after release

Always test updates in a non-production environment first and ensure you have current backups before upgrading.

What should I monitor on a daily basis?

Daily monitoring should include these key areas:

  • System Health: CPU, memory, disk space, and service status
  • Data Ingestion: Feed update status and data processing rates
  • Active Incidents: Open incidents requiring attention
  • Critical Alerts: High-priority security events and system alerts
  • Playbook Executions: Failed or stuck automated workflows
  • Integration Status: Third-party tool connectivity and health
  • User Activity: Login patterns and user access anomalies
  • Compliance Status: Critical control failures or gaps

Set up automated alerting for critical metrics to ensure timely response to issues.
