Threat Hunting Playbooks

1

APT Hunting

Proactive hunting for Advanced Persistent Threat indicators and behaviors.

# Example hunting query
SELECT * FROM network_connections 
WHERE destination_port IN (443, 80) 
AND connection_duration > 3600 
AND data_volume > 100MB
2

Insider Threat Detection

Detect anomalous user behavior that may indicate insider threats.

  • Unusual access patterns
  • Large data downloads
  • Off-hours activities
  • Privilege escalation attempts
3

Ransomware Hunting

Early detection of ransomware activity before encryption begins.

  • File encryption patterns
  • Suspicious process behavior
  • Network communication IOCs
  • Shadow copy deletion
Customization: All playbooks can be customized to match your organization's specific requirements and security policies.

API Reference

Comprehensive REST API documentation for integrating with Securaa platforms.

Authentication

API Security: All API requests must be authenticated using API keys or OAuth 2.0 tokens. 
# API Key Authentication
curl -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     https://api.securaa.com/v1/assets

Common Endpoints

Endpoint Method Description Platform
/v1/assets GET Retrieve asset inventory CSAM
/v1/alerts GET, POST Manage security alerts SIA
/v1/playbooks GET, POST Execute and manage playbooks SOAR
/v1/iocs GET, POST Manage indicators of compromise TIP
/v1/incidents GET, POST, PUT Incident management SOAR

API Examples

Retrieve Assets

GET /v1/assets
{
  "assets": [
    {
      "id": "asset-123",
      "name": "web-server-01",
      "type": "server",
      "ip_address": "192.168.1.100",
      "os": "Ubuntu 20.04",
      "criticality": "high",
      "last_scan": "2025-10-01T10:00:00Z"
    }
  ]
}

Create Alert

POST /v1/alerts
{
  "title": "Suspicious Network Activity",
  "severity": "high",
  "source": "192.168.1.100",
  "description": "Unusual outbound connections detected",
  "iocs": ["malicious-domain.com"]
}

Execute Playbook

POST /v1/playbooks/malware-response/execute
{
  "target_asset": "asset-123",
  "parameters": {
    "isolation_mode": "network",
    "scan_type": "full"
  }
}

Add IOC

POST /v1/iocs
{
  "type": "domain",
  "value": "malicious-domain.com",
  "threat_type": "malware",
  "confidence": 85,
  "source": "internal_analysis"
}

Webhook Configuration

Real-time Updates: Configure webhooks to receive real-time notifications for security events. 
# Webhook endpoint configuration
POST /v1/webhooks
{
  "url": "https://your-endpoint.com/webhook",
  "events": ["alert.created", "incident.updated"],
  "secret": "your-webhook-secret"
}
