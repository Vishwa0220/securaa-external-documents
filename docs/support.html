Frequently Asked Questions

Common questions and answers about Securaa products and services.

General Questions

What is Securaa and what does it offer?

Securaa is a comprehensive cybersecurity platform that provides four main solutions:

  • SIA (Security Intelligence Analytics): Advanced threat detection and analytics
  • SOAR (Security Orchestration): Automated incident response and orchestration
  • TIP (Threat Intelligence Platform): Threat intelligence collection and analysis
  • CSAM (Cyber Security Asset Management): Comprehensive asset management and vulnerability assessment

What are the system requirements for Securaa?

System requirements vary by solution:

Solution CPU RAM Storage
SIA 16+ cores 64+ GB 2+ TB SSD
SOAR 8+ cores 32+ GB 1+ TB SSD
TIP 8+ cores 32+ GB 1+ TB SSD
CSAM 12+ cores 48+ GB 1.5+ TB SSD

How long does it take to deploy Securaa?

Deployment time depends on the scope and complexity:

  • Proof of Concept (POC): 1-2 weeks
  • Single Solution: 2-4 weeks
  • Multiple Solutions: 4-8 weeks
  • Enterprise Deployment: 8-12 weeks

Our professional services team works with you to ensure smooth deployment and minimal disruption to your operations.

What integrations are available?

Securaa integrates with 200+ security tools including:

  • SIEM: Splunk, QRadar, ArcSight, Elastic Security
  • Endpoint: CrowdStrike, Microsoft Defender, Carbon Black
  • Cloud: AWS Security Hub, Azure Security Center, Google Cloud Security
  • Network: Palo Alto, Fortinet, Cisco, pfSense
  • Threat Intelligence: MISP, STIX/TAXII, OTX, VirusTotal

Technical Questions

How does Securaa handle data privacy and compliance?

Securaa is designed with privacy and compliance in mind:

  • Data Encryption: All data is encrypted in transit and at rest using AES-256
  • Access Controls: Role-based access control (RBAC) with multi-factor authentication
  • Compliance: Supports SOC 2, ISO 27001, GDPR, HIPAA, and other frameworks
  • Audit Logging: Comprehensive audit trails for all system activities
  • Data Residency: Options for on-premises, cloud, or hybrid deployments

Can Securaa be deployed in air-gapped environments?

Yes, Securaa supports air-gapped deployments with the following considerations:

  • Offline installation packages available
  • Manual threat intelligence feed updates
  • Local certificate management
  • Isolated network configurations supported
  • Custom update mechanisms for security patches

What backup and disaster recovery options are available?

Securaa provides comprehensive backup and disaster recovery features:

  • Automated Backups: Scheduled database and configuration backups
  • High Availability: Active-passive clustering for minimal downtime
  • Geo-Replication: Multi-site replication for disaster recovery
  • Point-in-Time Recovery: Restore to specific timestamps
  • Testing: Regular DR testing and validation procedures

Troubleshooting Guide

Common issues and their solutions for Securaa platforms.

Before You Begin: Always check the system logs and error messages for detailed information about any issues.

Installation Issues

Installation Fails with Database Connection Error

Symptoms: Installation stops with "Cannot connect to database" error

Possible Causes:

  • Database server not running
  • Incorrect database credentials
  • Firewall blocking database port
  • Database user lacks required permissions

Solution:

# Check database status
systemctl status postgresql

# Test database connection
psql -h localhost -U securaa -d securaa_db

# Check firewall rules
ufw status
iptables -L

# Grant database permissions
GRANT ALL PRIVILEGES ON DATABASE securaa_db TO securaa;

SSL Certificate Installation Fails

Symptoms: Web interface shows SSL certificate errors

Solution:

# Check certificate file permissions
ls -la /etc/ssl/certs/securaa.crt
ls -la /etc/ssl/private/securaa.key

# Verify certificate validity
openssl x509 -in /etc/ssl/certs/securaa.crt -text -noout

# Restart web service
systemctl restart securaa-web

Performance Issues

High CPU Usage

Symptoms: System becomes slow, high CPU utilization

Troubleshooting Steps:

# Check CPU usage by process
top -p $(pgrep securaa)

# Check system load
uptime

# Review recent log entries
journalctl -u securaa --since "1 hour ago"

# Check for runaway processes
ps aux | grep securaa | sort -k3 -nr

Common Solutions:

  • Increase CPU resources if possible
  • Optimize scan schedules to reduce overlap
  • Tune database query performance
  • Enable query caching

Memory Issues

Symptoms: Out of memory errors, system freezes

Diagnosis:

# Check memory usage
free -h
cat /proc/meminfo

# Check for memory leaks
valgrind --tool=memcheck --leak-check=full securaa-process

# Monitor memory usage over time
sar -r 1 60

Network Issues

Asset Discovery Not Working

Symptoms: New assets not being discovered automatically

Troubleshooting:

# Test network connectivity
nmap -sn 192.168.1.0/24

# Check SNMP connectivity
snmpwalk -v2c -c public 192.168.1.100

# Verify discovery credentials
ssh user@192.168.1.100 "echo 'Connection successful'"

# Check discovery logs
tail -f /var/log/securaa/discovery.log

Integration Issues

SIEM Integration Not Receiving Data

Symptoms: No data flowing to external SIEM

Troubleshooting Steps:

  1. Verify network connectivity to SIEM
  2. Check authentication credentials
  3. Validate data format and API endpoints
  4. Review integration logs for errors
  5. Test with sample data
# Test API connectivity
curl -X POST https://siem.company.com/api/events \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"test": "message"}'
